Antigravity Knowledge Work Kit - Sổ Tay Hướng Dẫn

Tài liệu hướng dẫn đầy đủ cho 15 Skills và 69 Workflows - giúp AI Agent trở thành cộng sự làm việc thực thụ cho từng phòng ban trong doanh nghiệp.

Tác giả: Minh Đỗ

PART I: NỀN TẢNG (FOUNDATION)

Chương 1: Giới thiệu

Antigravity Knowledge Work Kit là gì?

Antigravity Knowledge Work Kit là bộ công cụ giúp AI Agent làm việc chuyên nghiệp trong môi trường doanh nghiệp. Thay vì chỉ trả lời câu hỏi đơn giản, AI Agent được trang bị 15 Skills (kỹ năng chuyên môn) và 69 Workflows (quy trình thực thi) để xử lý các tác vụ phức tạp như:

  • Quản lý Sales pipeline (phễu bán hàng)
  • Triage (phân loại) ticket hỗ trợ khách hàng
  • Chuẩn bị legal briefs (tóm tắt pháp lý)
  • Phân tích financial variance (chênh lệch tài chính)
  • Viết feature specs (tiêu chuẩn tính năng sản phẩm)

Bố cục cuốn sách

Cuốn sách được chia thành 4 phần:

  • Part I - Nền tảng: Giới thiệu, kiến trúc hệ thống, hướng dẫn khởi động nhanh
  • Part II - Phòng ban: Hướng dẫn chi tiết theo từng phòng ban, mỗi chương là một phòng ban với đầy đủ Skills, Workflows và cách sử dụng thực tế
  • Part III - Nâng cao: Tùy chỉnh Skills, tạo Skills mới, quản lý hệ thống
  • Part IV - Phụ lục: Bảng tra cứu nhanh, bảng thuật ngữ, FAQ

Chương 2: Kiến trúc (Architecture)

3 thành phần cốt lõi

Antigravity Knowledge Work Kit xây dựng trên 3 thành phần chính:

1. Skills - Bộ kỹ năng chuyên môn mà AI Agent có thể sử dụng. Mỗi Skill gồm nhiều Sub-Skills (kỹ năng con) chuyên biệt.

2. Workflows - Quy trình thực thi từng bước, được kích hoạt qua slash commands (lệnh gạch chéo) như /sales-call-summary hoặc bằng ngôn ngữ tự nhiên trên Antigravity.

3. Connectors - Kết nối với các công cụ bên ngoài qua MCP (Model Context Protocol) servers như Slack, Gmail, Jira, Notion, Salesforce.

Cấu trúc thư mục


.agents/
├── skills/                    ← Thư mục chứa Skills
│   └── skill-name/
│       ├── SKILL.md           ← File chính (< 150 dòng)
│       ├── references/        ← Tài liệu chi tiết
│       └── scripts/           ← Script tiện ích
├── workflows/                 ← Thư mục chứa Workflows
│   └── workflow-name.md       ← Quy trình từng bước

3-Tier Progressive Disclosure (hệ thống tiết lộ dần 3 tầng)

Hệ thống được thiết kế theo nguyên tắc Progressive Disclosure (tiết lộ dần) để tối ưu token usage (lượng token sử dụng):

TầngFileMục đích
Tầng 1YAML frontmatterAgent đọc đầu tiên: tên, mô tả ngắn
Tầng 2SKILL.md (< 150 dòng)Hướng dẫn cốt lõi, bảng Sub-Skills, trigger conditions (điều kiện kích hoạt)
Tầng 3references/*.mdChi tiết chuyên sâu, chỉ đọc khi cần

Memory Management (Quản lý bộ nhớ)

Hệ thống Productivity Skill bao gồm cơ chế Memory Management giúp AI Agent nhớ ngữ cảnh làm việc. Trên nền tảng Antigravity, file GEMINI.md đóng vai trò hot cache (bộ nhớ nóng):


GEMINI.md          ← Hot cache (~30 người, ~30 thuật ngữ, dự án đang làm)
memory/
  glossary.md      ← Bảng giải mã thuật ngữ nội bộ đầy đủ
  people/          ← Hồ sơ chi tiết từng người
  projects/        ← Chi tiết dự án
  context/         ← Công ty, team, công cụ

GEMINI.md phủ khoảng 90% nhu cầu giải mã hàng ngày. Thư mục memory/ cung cấp độ sâu không giới hạn khi cần.

Chương 3: Quick Start (Bắt đầu nhanh)

Bước 1: Cấu trúc thư mục

Đảm bảo thư mục dự án của bạn có cấu trúc .agents/ đúng chuẩn:


your-project/
├── .agents/
│   ├── skills/        ← Chứa tất cả Skills
│   └── workflows/     ← Chứa tất cả Workflows
├── GEMINI.md          ← Hot cache cho Memory Management
└── TASKS.md           ← File quản lý công việc

Bước 2: Cấu hình Connectors

Mỗi Skill có thể kết nối với các công cụ bên ngoài qua MCP servers. Xem file references/connectors.md trong từng Skill để biết cách cấu hình. Các connector phổ biến:

LoạiCông cụ hỗ trợ
ChatSlack, Microsoft Teams
EmailMicrosoft 365, Gmail
CalendarMicrosoft 365, Google Calendar
Knowledge baseNotion, Confluence, Guru
Project trackerAsana, Linear, Jira, Monday, ClickUp
CRMSalesforce, HubSpot

Bước 3: Chạy Workflow đầu tiên

Trên Antigravity, bạn có thể kích hoạt Workflow bằng ngôn ngữ tự nhiên:

  • "Hãy khởi tạo hệ thống productivity cho tôi" → Chạy workflow productivity-start
  • "Tôi cần triage ticket này" → Chạy workflow customer-support-triage
  • "Viết feature spec cho tính năng mới" → Chạy workflow product-management-write-spec

Bước 4: Sử dụng Skills tự nhiên

Sau khi đã thiết lập, bạn chỉ cần nói chuyện bình thường với AI Agent. Agent sẽ tự động nhận diện Skill phù hợp dựa trên trigger conditions. Ví dụ:

  • "Review đoạn code này giúp tôi" → Agent kích hoạt Engineering > code-review
  • "Phân tích đối thủ cạnh tranh ABC" → Agent kích hoạt Marketing > competitive-analysis
  • "Chuẩn bị cho cuộc gọi bán hàng với công ty XYZ" → Agent kích hoạt Sales > call-prep

PART II: HƯỚNG DẪN THEO PHÒNG BAN

Chương 4: Phòng Năng suất và Tìm kiếm (Productivity & Enterprise Search)

Chương này dành cho tất cả nhân viên trong tổ chức, không giới hạn phòng ban cụ thể.

4.1 Productivity Skill

Quản lý công việc, lập kế hoạch ngày, và xây dựng bộ nhớ ngữ cảnh quan trọng. Đồng bộ với calendar, email, chat để tổ chức mọi thứ.

Mục đích: Biến AI Agent thành cộng sự làm việc hiểu được ngôn ngữ nội bộ của tổ chức.

Task Management (Quản lý công việc)

File TASKS.md trong thư mục dự án dùng để theo dõi công việc:


# Tasks
## Active
- [ ] **Hoàn thành báo cáo Q1** - cho sếp Hùng, due 15/03
## Waiting On
- [ ] **Feedback thiết kế** - chờ team Design, gửi 10/03
## Someday
- [ ] **Tối ưu quy trình CI/CD** - khi có thời gian
## Done
- [x] ~~Setup môi trường staging~~ (08/03)
Bạn nóiAgent làm
"Việc gì đang cần làm?"Đọc và tóm tắt Active + Waiting On
"Thêm việc mới" hoặc "nhớ làm"Thêm vào Active với format - [ ]
"Xong rồi" hoặc "hoàn thành"Đánh dấu [x], chuyển sang Done
"Nhớ rằng X là Y"Lưu vào memory
Memory Management (Quản lý bộ nhớ)

Hệ thống memory giúp Agent hiểu ngữ cảnh nội bộ mà không cần giải thích lại mỗi lần. Quy trình tra cứu:

1. Kiểm tra GEMINI.md (hot cache) → phủ hầu hết trường hợp

2. Tìm trong memory/glossary.md → bảng thuật ngữ đầy đủ

3. Kiểm tra memory/people/, memory/projects/ → chi tiết chuyên sâu

4. Hỏi người dùng → thuật ngữ chưa biết, rồi ghi nhớ lại

  • /productivity-update - Cập nhật và đồng bộ: đồng bộ tasks từ các nguồn, làm mới memory từ hoạt động gần nhất

Cách sử dụng thực tế

Tình huống 1: Chuẩn bị cuộc gọi bán hàng

  • Bạn nói: "Chuẩn bị cho cuộc gọi với công ty ABC ngày mai"
  • Agent kích hoạt call-prep: nghiên cứu công ty, tìm hiểu người tham dự trên LinkedIn, đề xuất agenda và talking points

Tình huống 2: Soạn email tiếp cận

  • Bạn nói: "Soạn email giới thiệu sản phẩm cho anh Hùng ở công ty XYZ"
  • Agent kích hoạt draft-outreach: nghiên cứu XYZ, tìm pain points phù hợp, soạn email cá nhân hóa

Tình huống 3: Review pipeline cuối tuần

  • Bạn nói: "Review pipeline của tôi tuần này"
  • Agent kích hoạt sales-pipeline-review: phân tích từng deal, gắn cờ rủi ro, đề xuất next actions

Viết feature specs, lập kế hoạch roadmap, tổng hợp user research nhanh hơn. Cập nhật stakeholders và theo dõi bối cảnh cạnh tranh.

Phụ lục C: FAQ (Câu hỏi thường gặp)

Có cần kết nối tất cả connectors mới dùng được không?

Không. Mỗi Skill đều hoạt động standalone (độc lập) với web search và input từ người dùng. Connectors giúp tăng sức mạnh nhưng không bắt buộc.

Tôi có thể tạo Skill mới cho phòng ban riêng không?

Có. Sử dụng Plugin Creator Skill (Chương 16) để tạo Skills tùy chỉnh theo quy trình đặc thù của tổ chức bạn.

Memory Management lưu ở đâu?

File GEMINI.md và thư mục memory/ nằm trong thư mục gốc của dự án. Đây là bộ nhớ cục bộ, không upload lên cloud.

Làm sao biết Skill nào đang được kích hoạt?

AI Agent sẽ tự động nhận diện Skill phù hợp dựa trên trigger conditions. Bạn cũng có thể gọi trực tiếp bằng tên Workflow (ví dụ: "chạy workflow sales-call-summary").

Sách này có áp dụng cho platform nào khác ngoài Antigravity không?
